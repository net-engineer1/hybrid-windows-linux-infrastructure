# Week 2 - Развертывание виртуальных машин

## Цель недели
Подготовить виртуальную среду для дальнейшего развертывания Active Directory.
На этом этапе домен НЕ настраивается

## Итог недели
- Установлен VirtualBox
- Создана VM с Windows Server
- Создана VM с Ubuntu Server
- ВМ запускаются и имеют сеть

## 1. Гипервизор

### Что используется
- VirtualBox

### Почему
- Бесплатный
- Подходит для учебных и тестовых сред
- Часто используется для лабараторий


## 2. Сетевая модель виртуальных машин

### Выбранная схема
- NAT - доступ в интернет
- Host-Only - внутренняя сеть между VM

### Почему именно так
- VM имеют доступ в интернет
- VM изолированы от внешней сети
- Соответствует реальной корпоративной схеме


## 3. Windows Server VM

### Назначение
- Будущий Domain Controller (AD, DNS, DHCP)

### Параметры VM
- OC: Windows Server
- CPU: 2
- RAM: 4 GB
- Disk: 50 GB

- Настройки сети/ настройки адаптера:
   - Adapter 1: NAT
   - Adapter 2: Host-Only (Виртуальный адаптер)


 ## 4. Ubuntu Server VM

 ### Назначение
 - Будущий сервер мониторинга / автоматизации

 ### Параметры VM
 - ОС: Ubuntu Server
 - CPU: 2
 - RAM: 2 GB
 - Disk: 20 GB
 
 - Сеть: 
    - Adapter 1: NAT
    - Adapter 2: Host-Only
    

## План работ:
- Создать Host-Only сеть VirtualBox
- Отключить DHCP в Host-Only
- Подключить NAT + Host-Only к Windows Server и Ubuntu Server
- Зафиксировать IP план

## Ip-план лаборатории:

```

| Устройство | Роль         | IP            |
|  Host      | Host-Only    | 192.168.56.1  |
|  DC01      | AD/DNS/DHCP  | 192.168.56.10 |
|  LNX01     | Linux Server | 192.168.56.20 |

````
